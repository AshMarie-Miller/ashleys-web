CLI and Git: A Basic Reference

March 5th, 2023

Whether you're a new developer, new to CLI and/or Git, or like me, you sometimes need a reminder of how to do something, this CLI and Git basic reference is for you.

What is CLI?

CLI stands for command-line interface, and it's a text-based UI (user interface) used to interact with your computer.

Basic CLI Reference
  • pwd:print working directory
  • ls:list directories
  • cd:change directory
  • mkdir:make directory
  • touch:create file
  • rm:delete file
  • rm -r:delete directory along with subdirectories and files
  • mv:move or rename files or directories
  • cp:copy files or directories
  • clear:clear your cli
  • man:manual pages

What is Git?

Git is a version control system used to track changes in files.

Basic Git Reference
  • git config:configure Git
  • git config --global user.name "First Last"
  • git config --global user.email username@email.com
  • git init:initialize Git repository
  • git status:check the status of a Git repository
  • git add:add changes in the working directory to the staging area
  • git commit:track files
  • git commit -m "Commit Message"
  • git push:upload files
  • git pull:download files
  • git clone:optain a repository from an existing URL
  • git diff:show the file differences which are not yet staged
  • git reset:unstages the file, but preserves the file contents
  • git rm:deletes the file from your working directory and stages the deletion
  • git log:lists the version history for the current branch
  • git show:shows the metadata and content changes of the specified commit
  • git tag:gives tags to the specified commit
  • git branch:lists all the local branches
  • git checkout:switches from one branch to another
  • git merge:merges the specified branch's history into the current branch
